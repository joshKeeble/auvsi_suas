Hardware Setup Check

Check that all connections are secure. The Ubuiqiti Nano Station and Rocket M5 should have two lights on in the back, the first for power and the second for connection to LAN. All ethernet ports should have blinking or solid LEDs on the switch. The battery for the pi should have full charge as is indicated by the LEDs next to the power button.

Ip Configuration Check

Check all ip connections and ports listed in the communication setup page through ping. It is good practice to use a different port for the connection to Mission Planner every instance due to an issue with how Mission Planner authorizes socket connections. If there are any changes to the port numbers or ip addresses, submit them through the networking tool in the side menu before mission activation. These settings will be saved to a file in /auvsi_suas/networking.txt.

Rules on available ports are on the documentation page and all ip addresses should be 192.168.1.x.

When pinging the uav payload Raspberry Pi, do not ping the Rocket M5, but instead the Raspberry Pi

Access /etc/network/interfaces to disable DHCP and set up a static ip address if the pi refuses to connect to the speficied etho or eno1 address on boot

PiCamera Check

Optional, but highly advised due to the uncertainity of whether the picamera is operational. SSH into the Payload Raspberry Pi through

`ssh pi@192.168.1.66`

If the ethernet ip was set correctly, check whether the picamera is operational through running:

`vcgencmd get_camera`

The output should be `supported = 1, detected = 1`.

If anything other than this, go into:

`sudo raspi-config`

to turn on the camera and if it still does not work, reboot. If still not working, check the integrity of the connection and ribbon cable. If all else, replace camera and start this step again.

Connect Mission Planner Server

If you haven't done so already, modify the paths appended to sys.path in mission_planner_main.py to match your computer. This is user specific.

First, ping the ground station from the Mission Planner laptop. Sometimes Windows firewall will make it such it will block connections from only one side

Under /Scripts in Mission Planner, load the python script from /auvsi_suas/python/src/main/mission_planner_main.py

After a few seconds, you should see a message saying that the server connected to the correct address. If not, the documentation will contain further instructions.

It is common to have to run with a different port if the port can't be opened.

Waypoint Capture

If all connections are working, go to Obstacle Avoidance section of the side menu and go to the stationary obstacles section. As of creating this program, we are not attempting moving obstacles.

It will display a list of waypoints for you to approve before being sent to Mission Planner

The UAV will travel to all waypoints and will loiter at the last waypoint until the next command, so pay attention to the waypoints and Mission Planner to know when to switch to the next mode.

Targeting System

There are two methods, manual and automatic.

For manual, you are going to select a directory of images and it will automatically parse them but will ask you to select and label the targets. It will estimate the location of the target and create objects and save them to a file.

Review the objects before being sent.

Upload the objects to the interoperability system.

Payload Deployment

There are again two methods, manual and automatic.

You can either verify the coordinates given by interop or enter coordinates

Select activate to send the coordinates to Mission Planner

Return to Base

Enter the GPS coordinates for home position.

Select activate to bring the UAV back to home position.

This function can also be used to send the UAV to a specific coordinate
